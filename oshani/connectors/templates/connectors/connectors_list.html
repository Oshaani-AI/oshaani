Manage your external service integrations

What are OAuth Connectors?

OAuth connectors allow you to securely connect your AI agents to external services like JIRA, Confluence, GitLab, GitHub, Google Workspace, and Microsoft 365. Once connected, you can automatically sync data from these services to train your agents, enabling them to answer questions about your projects, documentation, and workflows.

How to Configure OAuth Connectors

Follow these steps to set up each connector type:

  1. Go to Atlassian Developer Console
  2. Click "Create" → "OAuth 2.0 integration"
  3. Fill in the application details:
    • Name: Your app name (e.g., "Oshani AI Agent")
    • Application type: Web application
    • Authorization callback URL: Copy the callback URL shown after creating the connector
  4. Add the following scopes:
    • read:jira-work - Read issues and work items
    • read:jira-user - Read user information
    • read:issue-details:jira - Read issue details
  5. Copy the Client ID and Client Secret
  6. Create the connector in Oshani with:
    • Base URL: Your JIRA instance URL (e.g., https://your-domain.atlassian.net)
    • Client ID: From Atlassian Developer Console
    • Client Secret: From Atlassian Developer Console
  7. Click "Connect" to authorize the application

  1. Go to Atlassian Developer Console
  2. Click "Create" → "OAuth 2.0 integration"
  3. Fill in the application details:
    • Name: Your app name
    • Application type: Web application
    • Authorization callback URL: Copy the callback URL shown after creating the connector
  4. Add the following scopes:
    • read:confluence-content.all - Read all Confluence content
    • read:confluence-space.summary - Read space summaries
  5. Copy the Client ID and Client Secret
  6. Create the connector with your Confluence instance URL
  7. Click "Connect" to authorize

  1. Go to your GitLab instance → User Settings → Applications
  2. Or for GitLab.com: https://gitlab.com/-/profile/applications
  3. Click "New application"
  4. Fill in the details:
    • Name: Your application name
    • Redirect URI: Copy the callback URL shown after creating the connector
    • Scopes: Select read_api and read_repository
  5. Click "Save application"
  6. Copy the Application ID (Client ID) and Secret
  7. Create the connector with:
    • Base URL: Your GitLab instance URL (e.g., https://gitlab.com or https://gitlab.yourcompany.com)
    • Client ID: Application ID from GitLab
    • Client Secret: Secret from GitLab
  8. Click "Connect" to authorize

  1. Go to GitHub → Settings → Developer settings → OAuth Apps
  2. Or visit: https://github.com/settings/developers
  3. Click "New OAuth App"
  4. Fill in the details:
    • Application name: Your app name
    • Homepage URL: Your website URL
    • Authorization callback URL: Copy the callback URL shown after creating the connector
  5. Click "Register application"
  6. Copy the Client ID and generate a Client Secret
  7. Create the connector with:
    • Base URL: https://api.github.com (or your GitHub Enterprise URL)
    • Client ID: From GitHub
    • Client Secret: From GitHub
  8. Click "Connect" to authorize

  1. Go to Google Cloud Console
  2. Select or create a project
  3. Navigate to APIs & Services → Credentials
  4. Click "Create Credentials" → "OAuth client ID"
  5. If prompted, configure the OAuth consent screen first:
    • Choose External user type
    • Fill in app information
    • Add scopes: https://www.googleapis.com/auth/drive.readonly, https://www.googleapis.com/auth/documents.readonly
  6. Create OAuth client:
    • Application type: Web application
    • Name: Your app name
    • Authorized redirect URIs: Copy the callback URL shown after creating the connector
  7. Copy the Client ID and Client Secret
  8. Create the connector with:
    • Base URL: https://www.googleapis.com
    • Client ID: From Google Cloud Console
    • Client Secret: From Google Cloud Console
  9. Click "Connect" to authorize

  1. Go to Azure Portal
  2. Click "New registration"
  3. Fill in the details:
    • Name: Your application name
    • Supported account types: Choose based on your needs
    • Redirect URI: Web → Copy the callback URL shown after creating the connector
  4. Click "Register"
  5. Go to "API permissions" and add:
    • Files.Read.All - Read all files
    • Sites.Read.All - Read all site collections
    • User.Read - Sign in and read user profile
  6. Click "Grant admin consent" if required
  7. Go to "Certificates & secrets" → Create a new client secret
  8. Copy the Application (client) ID and Client secret value
  9. Create the connector with:
    • Base URL: https://graph.microsoft.com
    • Client ID: Application ID from Azure
    • Client Secret: Client secret from Azure
  10. Click "Connect" to authorize
Use Cases for Connectors
JIRA Connector
  • Train agents on completed issues and resolutions
  • Answer questions about project status and workflows
  • Provide insights from historical issue data
  • Help with ticket creation and management
Confluence Connector
  • Train agents on company documentation
  • Answer questions from knowledge base articles
  • Provide context from internal wikis
  • Help with documentation search and retrieval
GitLab/GitHub Connectors
  • Train agents on code repositories and README files
  • Answer questions about project structure
  • Provide context from pull requests and issues
  • Help with code documentation and examples
Google/Microsoft Connectors
  • Train agents on documents and files
  • Answer questions from shared drives
  • Provide context from company documents
  • Help with document search and information retrieval
How It Works
  1. Create: Set up your connector with OAuth credentials
  2. Connect: Authorize the connector via OAuth (you'll be redirected to the service provider)
  3. Sync: Sync data from the connected service to your agent's training data
  4. Train: Use the synced data to train your AI agent
  5. Query: Your agent can now answer questions based on the synced data
